# Nexarch 3D City — Pipeline & Handoff Notes

How the Nexarch city view (`nexarch-3d.html`) works, where every file lives,
and what an AI-styling pass (Higgsfield or similar) must and must not touch.

## Architecture in one paragraph

The city uses the Diablo 2 approach: the player sees a **pre-rendered
image** (baked in Blender Cycles from one fixed isometric camera), while
invisible 3D geometry fills the depth buffer so live 3D agents walking the
city are correctly hidden behind buildings. Agents pathfind on a walk grid.
Everything is procedurally generated by the scripts in
`scripts/blender/nexarch-kit/` — nothing is hand-modeled.

## The files ("where is the 3D world?")

| File | What it is |
|---|---|
| `images/3d/nexarch-bake.jpg` | **The look of the city.** 6144x4200 Cycles render, fixed camera. This is the file to restyle/upscale/animate. |
| `images/3d/tiles/` | The same image sliced into a 3-level zoom pyramid (L0 overview + 512px L1/L2 tiles). This is what the page actually loads. Regenerated from the bake by `gen_tiles.py`. |
| `images/3d/nexarch-kit.glb` | The 3D building kit (36 pieces: houses, cathedral, arena, walls, kiosks, props, NPCs). Used for depth/occlusion and the live-3D fallback mode. |
| `images/3d/nexarch-layout.json` | Placements of all ~850 pieces, terrain heightfield, walkability grid, district labels. Coordinates match `world-coordinates.js` (city center = world 10500,9000). |
| `images/3d/nexarch-ground.png` | Cobblestone/terrain ground texture used by live mode and the bake. |
| `nexarch-3d.html` | The viewer. Cinematic mode (baked tiles + live agent) by default; `V` toggles live 3D. |
| `scripts/blender/nexarch-kit/` | Regenerates everything: `gen_kit_textures.py` → `build_kit.py` → `gen_layout.py` → `bake_city.py` → `gen_tiles.py` (run with `pip install bpy pillow`, then `python3 <script>`; they read/write `/tmp/citybuild`). |

## Camera lock (do not change)

The bake camera and the web camera must stay identical or agent occlusion
breaks: orthographic, yaw 225° (three.js) / azimuth 135° (Blender),
pitch 38°, ortho width 830 world units, target (0,0,0), aspect 6144:4200.

## Rules for an AI restyling pass (img2img)

1. **Geometry must not move.** Use image-to-image with the bake as input at
   low/medium strength. Building silhouettes, street courses and the city
   outline must stay pixel-aligned — the depth buffer and walk grid assume
   the image matches the geometry. Test on one 1024px crop at 2–3 strengths
   and compare alignment before processing the full image.
2. **Style target:** dark medieval cyberpunk, painted isometric game-art
   look. Black gabled rooftops, beveled stone, glowing cyan/magenta circuit
   windows, neon panel roofs, warm orange doorways, wet cobblestone with
   puddle reflections, rain mood. Reference: the AI concept art in
   `images/` and the Lovable map. No neon lines on the ground itself —
   streets are plain wet cobble.
3. **Process in overlapping regions** (the image is 6144 wide; image models
   cap around 2048 — feeding the whole map gives a restyled center crop,
   not the city). Use the ready-made tooling:
   ```
   python3 scripts/blender/nexarch-kit/restyle_tools.py split images/3d/nexarch-bake.jpg /tmp/restyle
   # img2img every /tmp/restyle/in/*.png -> /tmp/restyle/out/<same name>.png
   #   (12 regions, SAME prompt/strength/settings for all)
   python3 scripts/blender/nexarch-kit/restyle_tools.py stitch /tmp/restyle images/3d/nexarch-bake.jpg
   ```
   The stitcher feather-blends the 256px overlaps; keep global color
   grading consistent across regions (identical settings, ideally same
   seed).
4. **Output:** one finished 6144x4200 image saved over
   `images/3d/nexarch-bake.jpg`, then re-run `gen_tiles.py` (PIL only, no
   Blender needed) to rebuild `images/3d/tiles/`, and verify
   `nexarch-3d.html` still aligns (toggle `V` and compare live vs baked at
   the same spot — the silhouettes should overlap).
5. The full-city image can also be used as the input for ambient video
   loops (rain/fog/neon flicker) — that's an additive layer, not a
   replacement; talk to the user before changing the viewer for it.
